## Ce qui a été construit — `python\subnet_projet.py`
Programme complet et fonctionnel :
```python
def hotes_utilisables(prefixe):
    return 2 ** (32 - prefixe) - 2

prefixe = int(input("Ecris un nombre entre 0 et 32: "))
while prefixe < 0 or prefixe > 32:
    print("Ce nombre n'est pas entre 0 et 32")
    prefixe = int(input("Ecris un nombre entre 0 et 32: "))

print(prefixe, "->", hotes_utilisables(prefixe))

if prefixe <= 24:
    print("Grand reseau")
elif prefixe <= 28:
    print("Reseau moyen")
else:
    print("Petit reseau")
```

## Concepts pratiqués (dont nouveaux)
- **`while` en validation** (« redemander tant que la saisie est invalide ») — enfin pratiqué, réussi.
- **`elif`** — écrit pour la 1re fois par l'élève, réussi.
- **`and` vs `or`** : gros point. A d'abord écrit `while prefixe < 0 and prefixe > 32` (impossible → boucle jamais exécutée). Compris le basculement : **valide/dedans = `and`, invalide/dehors = `or`**.
- **`else` ne prend pas de condition** : a écrit `else prefixe > 28:` puis **repéré le problème LUI-MÊME** (ligne 18). Corrigé en `else:`.
- **`&` n'est pas le `and` logique** (opérateur binaire) — même famille de piège que `^` ≠ puissance. Corrigé.
- Élégance du `if`/`elif` : chaque branche profite du fait que les précédentes sont fausses → `elif prefixe <= 28` suffit (pas besoin de `>= 25`).
